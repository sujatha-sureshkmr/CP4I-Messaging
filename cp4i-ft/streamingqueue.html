Configure Streaming Queue

  1. Overview

Kafka Connect with IBM Event Streams allows you to seamlessly integrate Kafka with various data sources and sinks, enabling real-time data streaming and data synchronization between different systems in your architecture. It simplifies the process of building and managing data pipelines, making it easier to work with data in a Kafka-based environment.

We will now setup and configure streaming queue.


  1. EventStreams

We will now access eventstreams.

  1. Go to bastion server -> login to ocp
  2. Execute the following command copy the route from oc get routes output
    3. oc patch configs.imageregistry.operator.openshift.io/cluster --patch '{"spec":{"defaultRoute":true}}' --type=merge 
                                    
    oc get routes -n openshift-image-registry -o custom-columns=:.spec.host --no-headers
  3. Go to ocp console -> workloads -> secret -> event-admin -> copy the password
  4. Go to CloudPak for Integration console -> Integration Instances -> click "ft-eventstreams"

  1. Generate SCRAM credentials for eventstreams

We will now generate SCRAM credentials

  1. use username as "event-admin" and password (use the password retreieved from secret)
  2. Click "Connect to this cluster"
  3. Go to Internal -> Click "Generate SCRAM credentials"
  4. Use credential name "ft-evs-int-2", choose "Produce messages, consume messages and create topics and schemas". Click next.
  5. Choose "All Topics". Click next.
  6. Choose "All Consumer Groups". Click next.
  7. Choose "All Transaction ID". Click Generate Credentials.
  8. You will find the credentials generated.
  9. You will also find the credentials created in the secret.
  10. Go to External -> Click "Generate SCRAM credentials". Name: "ft-evs-cred-ext",Produce messages: Choose "All Topics", consume messages : "All Consumer Groups", create topics and schemas : "All Transaction ID", "Generate Credentials".
  11. Go to ocp console -> kube:admin -> copy login command
  12. Copy API token from the login command details

  1. KafkaConnect

We will now create Kafka Connect.

  1. Go to Workloads -> secret -> project (cp4i) -> Create image pull secret -> update following information -> click save
    2. oc get routes -n openshift-image-registry -o custom-columns=:.spec.host --no-headers
    Option Value Remarks
    Secret name cp4i-image-stream
    Registry server address paste the output of the oc get routes for openshift-image-registry Example: default-route-openshift-image-registry.apps.daffy-ak48g5wg.cloud.techzone.ibm.com
    Username oc Example: sha256~B0jEABlqy1TGZqJCrJ1uxLgjenL_37m9Kp0oB2wFj1Y
    Password paste API token
  2. Go to ocp console -> Operators -> Installed Operators -> IBM Event Streams -> kafka connect -> create kafka connect -> Yaml view -> paste the yaml code before
  3. Note: you will need to change output -> image variable from oc get routes. For example: 
    4. oc get routes -n openshift-image-registry -o custom-columns=:.spec.host --no-headers
  4. check 1: verify build config created. Go to Builds -> BuildsConfigs
  5. check 2: verify build created and completed successfully. Go to Builds -> Builds
  6. check 3: pods are created and successfully running. Note: This would take 3 to 5 mins to complete. Go to workloads -> pods.
  7. check 4: kafka connect container created and running successfully. Go to Operators -> Installed Operators -> IBM Event Streams -> Kafka Connect.
  8. Kafka connector Go to Operators -> Installed Operators -> IBM Event Streams -> Kafka Connector -> Create Kafka Connector -> yaml view -> update the code given below
  9. Kafka connector is successfully created.
